= Net32 Dental Supply — Spring Boot 3.x Microservices Migration Demo
:toc:
:toc-placement: preamble

This repository is a Blitzy migration demo: a representative JBoss EAP 8.0 / Jakarta EE 10
codebase re-platformed into three independently deployable Spring Boot 3.x microservices.

== What this is

The `kitchensink` module was a single WAR application deployed on JBoss EAP 8.0, with all
business logic living in six PL/pgSQL stored procedures. Blitzy decomposed it into three
independently deployable Spring Boot 3.5.16 / Java 17 microservices organized by business domain:

[cols="1,1,3"]
|===
|Service |Port |Bounded context

|`marketplace-service`
|`:8081/marketplace`
|Catalog, vendors, pricing — `PricingService`, `VendorSelectionService`

|`orders-service`
|`:8082/orders`
|Carts, orders, shipping, discounts — `OrderService`, `ShippingService`, `DiscountService`

|`users-service`
|`:8083/users`
|Members, loyalty-tier recalculation — `MemberRegistration`, `TierRecalculationService`
|===

All six stored procedures are re-implemented in pure Java (zero `createNativeQuery` calls).
All 54 automated tests pass: 37 `@SpringBootTest` + Testcontainers integration tests and 17
Playwright end-to-end tests driving the PHP storefront in a real browser.

== Repository layout

----
kitchensink/               <- Maven aggregator (packaging=pom)
├── marketplace-service/   <- Spring Boot, port 8081
├── orders-service/        <- Spring Boot, port 8082
├── users-service/         <- Spring Boot, port 8083
├── e2e/                   <- Playwright end-to-end suite
├── frontend/              <- PHP storefront (unchanged contract)
└── db/                    <- Read-only reference SQL (schema, procs, seed)
----

== Getting started

Full build, run, test, and Docker instructions are in
link:kitchensink/README.adoc[kitchensink/README.adoc].

Quick start:

[source,bash]
----
cd kitchensink
mvn clean install          # builds all 3 modules + runs Testcontainers ITs (Docker required)
----
